3. Arquitectura — qué construiste (diagrama)

Flujo en palabras simples: ArcGIS/Excel alimentan el sistema → Python limpia y exporta → Mapbox sirve geometrías pesadas → JSON/SQLite sirven atributos y búsqueda → FastAPI une todo → el navegador muestra mapa + analítica.

4. Conceptos educativos (glosario visual)

Shapefile (.shp)
Formato GIS clásico: geometría + atributos. Tus 22 capas (predios, barrios, vías…) son la fuente de verdad espacial.
Tileset vectorial (Mapbox)
Mapbox convierte millones de vértices en teselas pequeñas. El navegador solo carga lo visible en pantalla → por eso 140K predios se ven fluidos.
Mapbox GL JS
Librería JavaScript que pinta mapas GPU-acelerados en el browser. Reemplaza plugins pesados de Leaflet para datasets grandes.
Turf.js
“Excel espacial” en JavaScript: distancias, áreas, punto más cercano. Usado en herramientas de medición y rutas.
ETL (Extract-Transform-Load)
Tus scripts export_data.py y export_predios_full.py extraen Excel/SHP, transforman y cargan JSON/SQLite.
Ficha Predial 360°
Vista que cruza catastro + densidad + retiros + uso. Replica lógica DAX de Power BI en JavaScript.
FastAPI
Backend Python moderno. Expone REST API: búsqueda de predios, KPIs, filtros por barrio.
SQLite / PostGIS
SQLite: base embebida (archivo predios.db). PostGIS: PostgreSQL con funciones espaciales (Docker local, producción enterprise).
Render
Plataforma cloud que ejecuta tu Python 24/7 con URL pública. Plan free se duerme; demo sigue siendo válida.
Analytics Engineer vs Geospatial Engineer
AE: modela datos, métricas, pipelines en warehouse. Geo: geometrías, mapas, análisis espacial. Tú hiciste ambos en un producto.
